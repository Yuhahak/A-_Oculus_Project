- 서브 클래싱과 메시지 반사의 개념(누가 메시지를 처리하는가?)
서브 클래싱 - 윈도우 프로시저로 가는 메시지를 중간에 붙잡아 처리하는 기법 
-> 윈도우 동작을 바꿀 수 있음

메시지 반사 - 부모 윈도우가 처리하지 않는 통지 메시지를 컨트롤 자신이 처리할 수 있게
 MFC에서 제공하는 기법

- 서브 클래싱과 메시지 반사 구현 순서
서브클래싱 순서
1. 새로운 윈도우나 컨트롤 클래스 생성
2. 메시지 핸들러를 추가해서 기능을 재정의 -> 정적 서브클래싱
3. CWnd::SubclassWindow(), CWnd::SubclassDlgItem()함수 호출 -> 동적 서브클래싱

메시지 반사 구현 순서
1. 기존 컨트롤 클래스를 상속받아 새로운 클래스를 만들고
자신의 통지 메시지를 처리하는 함수 작성
2. 새로 만든 클래스를 이용하여 컨트롤 생성

- 대화 상자의 개념과 종류
대화 상자 = 입력을 받거나 정보를 출력하는 윈도우
종류 = 모드형 -> 대화상자를 닫아야 다른 작업가능
            비모드형 -> 닫지 않아도 작업가능
